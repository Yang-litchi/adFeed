# 统计数据持久化修复报告

## 概述

合并 `webLayer` 和 `with_statistics` 两分支时发现：`with_statistics` 分支实现的统计功能（点击量、曝光量）在 App 重启后全部清零。本文档记录问题根因、修复过程及修复后的接口架构。

---

## 问题描述

- **现象**：App 运行期间点击和曝光统计正常显示，但重启 App 后所有统计数据归零
- **影响范围**：点击量、曝光量、点赞状态、收藏状态全部丢失
- **根因**：`with_statistics` 分支未接入 Room 数据库，所有数据仅存于内存

---

## 两分支差异对比

| 模块 | `webLayer` | `with_statistics` |
|------|-----------|-------------------|
| Room 数据库 (`AppDatabase`) | ✅ 完整 | ❌ 不存在 |
| Room/KSP 依赖 | ✅ `build.gradle.kts` | ❌ 完全缺失 |
| `AdApplication`（Room 初始化） | ✅ | ❌ 不存在 |
| `InteractionEntity` + DAO | ✅ 持久化到 Room | ❌ `HashMap<String, LocalAdState>` 内存 |
| `AICacheEntity` + DAO | ✅ 持久化到 Room | ❌ 不存在 |
| 曝光/点击统计模型 | ❌ | ✅ `StatisticEvent` 等 |
| 统计接口 (`StatisticsRepository`) | ❌ | ✅ |
| 统计实现 | ❌ | `FakeStatisticsRepository` — **纯内存** |
| 曝光追踪 (`ExposureTracker`) | ❌ | ✅ 会话级内存（有意设计） |
| 统计 UI (`StatisticsScreen`) | ❌ | ✅ |

### 内存存储位置（重启丢失的四个数据源）

```
1. 统计事件（点击+曝光）
   → FakeStatisticsRepository.events = CopyOnWriteArrayList<StatisticEvent>()
   → App 进程被杀 → events 清空 → 全部丢失

2. 点赞/收藏状态
   → FeedViewModel.localStates = HashMap<String, LocalAdState>()
   → ViewModel 随 Activity 销毁 → 全部丢失

3. 曝光去重标记
   → ExposureTracker.exposedAdIds = MutableSet<String>()
   → 会话级内存，设计上正确（每次会话允许重新曝光）

4. 计数显示
   → AdItem.clickCount / AdItem.exposureCount
   → AdItem 来自 MockData，每次重新构建，从 0 开始
```

## 修复内容

### 修复策略

```
                 webLayer（有 Room，无统计）         with_statistics（有统计，无 Room）
                           │                                  │
                           └────────────┬─────────────────────┘
                                        ▼
                              merge_stats_fix（Room + 统计持久化）
              1. 从 webLayer 合并 Room 基础设施
              2. 新增 StatisticEventEntity + DAO
              3. 新建 RoomStatisticsRepository 实现 StatisticsRepository 接口
              4. FeedViewModel 接入 Room DAO + RoomStatisticsRepository
              5. StatisticsViewModel 默认使用 RoomStatisticsRepository
```

### 文件变更清单

#### 构建配置（3 文件修改）

| 文件 | 变更 |
|------|------|
| `gradle/libs.versions.toml` | 添加 KSP 版本 `2.2.10-2.0.2` 和 `ksp` 插件声明 |
| `build.gradle.kts`（根） | 注册 KSP 插件 `apply false` |
| `app/build.gradle.kts` | 应用 KSP 插件；添加 Room 依赖；启用 `buildConfig` |

#### Room 基础设施 — 从 webLayer 合并（5 文件新增）

| 文件 | 说明 |
|------|------|
| `AdApplication.kt` | Room 数据库初始化，`companion object` 暴露 `database` 单例 |
| `data/local/entity/InteractionEntity.kt` | 点赞/收藏持久化实体（主键 `adId`） |
| `data/local/entity/AICacheEntity.kt` | AI 文案缓存实体 |
| `data/local/dao/InteractionDao.kt` | 交互状态 DAO（增改 + Flow 订阅） |
| `data/local/dao/AICacheDao.kt` | AI 缓存 DAO |

#### 统计持久化 — 新建（3 文件新增）

| 文件 | 说明 |
|------|------|
| `data/local/entity/StatisticEventEntity.kt` | 统计事件实体（自增主键 `id`，`adId`，`tags`，`eventType`，`timestamp`） |
| `data/local/dao/StatisticEventDao.kt` | 统计查询 DAO（事件写入、曝光/点击计数、Tag 筛选、每日趋势聚合） |
| `data/repository/RoomStatisticsRepository.kt` | `StatisticsRepository` 接口的 Room 实现，替代内存版 |

#### 核心逻辑修改（4 文件修改）

| 文件 | 变更 |
|------|------|
| `AppDatabase.kt` | 版本升至 2，注册 `StatisticEventEntity` + `StatisticEventDao` |
| `FeedViewModel.kt` | 用 `InteractionDao` 替代 `HashMap`；用 `RoomStatisticsRepository` 替代 `FakeStatisticsRepository`；`loadAds()` 从 Room 恢复曝光/点击计数 |
| `StatisticsViewModel.kt` | 默认仓库改为 `RoomStatisticsRepository` |
| `QwenApi.kt` | API Key 改用 `BuildConfig.QWEN_API_KEY` |
| `AndroidManifest.xml` | 注册 `android:name=".AdApplication"` |

## 修复后的接口架构

### 接口层级一览

```
┌─────────────────────────────────────────────────────────┐
│                      业务接口层                          │
│                                                         │
│  StatisticsRepository (interface)                       │
│  ├── recordExposure(event)                              │
│  ├── recordClick(event)                                 │
│  ├── getAdStatistics(adId): StatisticSummary            │
│  ├── getTagStatistics(tag): List<TagStatistic>          │
│  └── getTrendStatistics(adId): List<TrendData>          │
│         ↑                          ↑                    │
│         │                          │                    │
│  FakeStatisticsRepository    RoomStatisticsRepository   │
│  (内存实现，可用于测试)       (Room 实现，生产环境)       │
│                                    │                    │
│                                    └── StatisticEventDao│
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                     Room DAO 层                          │
│  （接口由 @Dao 注解声明，实现由 KSP 编译时自动生成）       │
│                                                         │
│  InteractionDao           StatisticEventDao             │
│  ├── getInteraction()     ├── insertEvent()             │
│  ├── getAllFlow()         ├── getExposureCount()        │
│  ├── getSingleFlow()      ├── getClickCount()           │
│  └── insertOrUpdate()     ├── getAdIdsByTag()           │
│                            ├── getDailyEventCounts()    │
│  AICacheDao               └── getAllEventsFlow()        │
│  (AI缓存增删查)                                          │
└─────────────────────────────────────────────────────────┘
```

### ViewModel 依赖关系

```
FeedViewModel ─────────────┬──→ StatisticsRepository (接口) ──→ RoomStatisticsRepository
                           │                                         │
                           ├──→ InteractionDao ──→ Room DB           │
                           │                                  StatisticEventDao
                           │                                         │
                           └── observeDatabaseChanges()              ▼
                               订阅 Flow 自动同步 UI              Room DB
                               (interactions 表)

StatisticsViewModel ───────→ StatisticsRepository (接口) ──→ RoomStatisticsRepository
                                                                    │
                                                              StatisticEventDao
                                                                    │
                                                                    ▼
                                                               Room DB
                                                          (statistic_events 表)
```

### 设计优点

1. **接口隔离**：ViewModel 只依赖 `StatisticsRepository` 接口，不感知底层是内存还是 Room
2. **可替换性**：切换实现只需改一处默认值，无需修改 ViewModel 逻辑
   ```kotlin
   // 生产 → 测试 只需一行
   class StatisticsViewModel(
       private val repository: StatisticsRepository = RoomStatisticsRepository
       //                                           ^^^^^^^^^^^^^^^^^^^^^^^^^^^^
   )
   ```
3. **向后兼容**：`FakeStatisticsRepository` 保留，编写单元测试时可直接注入
4. **DAO 层单一职责**：三个 DAO 各管一张表，互不干扰

## 修复后的数据流

### 用户点赞（持久化 + 实时同步）

```
用户点击点赞
  ↓
FeedViewModel.toggleLike(adId)
  ↓
InteractionDao.insertOrUpdate(entity)  →  Room DB (interactions 表)
  ↓
Room Flow 推送变更
  ↓
observeDatabaseChanges()  →  FeedUiState.update()  →  UI 自动刷新
```

### 曝光/点击记录（持久化 + 自恢复）

```
┌── 写入路径 ──────────────────────────────────────┐
│                                                  │
│  ExposureDetector / onCardClick                  │
│    ↓                                             │
│  FeedViewModel.recordExposure/recordClick()      │
│    ├─→ UI State 立即更新 (clickCount + 1)        │
│    └─→ RoomStatisticsRepository.recordXxx()      │
│          └─→ StatisticEventDao.insertEvent()     │
│                └─→ Room DB (statistic_events 表)  │
└──────────────────────────────────────────────────┘

┌── 恢复路径（App 重启） ──────────────────────────┐
│                                                  │
│  FeedViewModel.init()                            │
│    └─→ loadAds()                                 │
│          ├─→ InteractionDao.getInteraction()      │
│          │     恢复点赞/收藏状态                   │
│          ├─→ StatisticEventDao.getExposureCount() │
│          │     恢复曝光计数                        │
│          └─→ StatisticEventDao.getClickCount()     │
│                恢复点击计数                        │
└──────────────────────────────────────────────────┘
```

### Room 数据库表结构

**表 `interactions`**（来自 webLayer）

| 列 | 类型 | 说明 |
|----|------|------|
| adId | TEXT (PK) | 广告 ID |
| isLiked | INTEGER | 是否点赞 |
| isCollected | INTEGER | 是否收藏 |
| likeCountOffset | INTEGER | 用户操作引起的点赞偏移量 |
| collectCountOffset | INTEGER | 用户操作引起的收藏偏移量 |
| shareCount | INTEGER | 分享统计数 |

**表 `statistic_events`**（新增）

| 列 | 类型 | 说明 |
|----|------|------|
| id | INTEGER (PK, 自增) | 事件 ID |
| adId | TEXT | 广告 ID |
| tags | TEXT | 标签列表，逗号分隔 |
| eventType | TEXT | `"EXPOSURE"` 或 `"CLICK"` |
| timestamp | INTEGER | 事件时间戳（毫秒） |
