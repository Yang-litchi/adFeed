## 1. 模块划分总览

### 1.1 模块全景

```
AI广告推荐信息流 App（全本地数据，无网络依赖）
│
├── app           ← 应用壳（装配层）：手工DI容器、导航路由、Application初始化
│
├── core          ← 基础核心层：本地文件读取、图片加载(Coil)、日志(Timber)、Base类、通用工具
│   └── player    ← 播放器：ExoPlayer封装、播放器池（≤2实例）、生命周期管理
│
├── data          ← 数据层：数据模型、Repository实现、本地JSON加载、Room DAO、缓存策略
│
├── feed          ← 信息流：RecyclerView列表、3种卡片ViewType、Tab切换、刷新/加载更多
├── detail        ← 详情页：图文详情、视频详情、互动UI(点赞彩蛋/收藏/分享)
├── analytics     ← 埋点统计：曝光/点击追踪、统计Room持久化、可视化图表
│
└── ai            ← AI增强（可选）：本地AI预生成数据加载、标签过滤、对话式搜索
```

### 1.2 模块与 Clean Architecture 层级映射

| 模块 | Clean Architecture 层级 | 数据依赖 | 可并行开发 |
|------|------------------------|---------|-----------|
| `app` | 装配层 | 所有模块 | ❌（最后装配） |
| `feed` / `detail` | Presentation + Domain | `data`（接口）、`core` | ✅ 相互独立 |
| `ai` | Domain + Data（可选） | `data`、`core` | ✅ 独立 |
| `data` | Data | `core` | ✅ 独立 |
| `player` | Infrastructure | `core` | ✅ 独立 |
| `core` | Infrastructure | 无（最底层） | ✅ 独立 |
| `analytics` | Infrastructure + Presentation | `data`、`core` | ✅ 独立 |

### 1.3 模块间通信方式

| 通信类型 | 机制 | 示例 |
|---------|------|------|
| 同步调用 | 接口 + 手工DI注入 | `FeedViewModel` 调用 `AIAdRepository.getEnhancedFeed()` |
| 跨页面数据传递 | Navigation SafeArgs | Feed → Detail 传递 `adId`、`channelId` |
| 跨页面状态同步 | `SharedFlow` 事件广播 | 详情页点赞 → 信息流卡片状态同步 |
| 异步通知 | `StateFlow` 状态观察 | 数据刷新 → UI 自动更新 |
| 可选功能解耦 | 接口 + 统一降级 | AI 数据缺失时 `AIAdRepositoryImpl` 内部降级为 description 截断 |

## 2. 模块1：app — 应用壳工程

### 2.1 模块职责

- 提供 `Application` 子类，初始化全局组件（Timber 日志、Coil 图片加载器、Room 数据库）
- 通过 **手工 DI 容器（AppContainer）** 创建并管理各模块的依赖实例
- 提供 `MainActivity` 作为唯一 Activity，承载 Navigation 导航
- 管理全局导航路由
- 统一的用户提示（Toast/SnackBar）

> **设计意图**：app 是唯一的"知道一切"的模块。手工 DI 容器让依赖关系一目了然，新手可读可调试。

### 2.2 对外接口

app 模块不对外暴露接口（它是顶层消费方）。它通过 `AppContainer` 向其他模块提供依赖实例。

```kotlin
// AppContainer — 手工 DI 容器（唯一的装配点）
class AppContainer(private val context: Context) {

    // ===== 基础设施 =====
    private val logService: LogService = LogServiceImpl()
    private val dispatcherProvider: DispatcherProvider = CoroutineDispatcherProvider()
    val imageLoader: ImageLoader = ImageLoaderProvider.create(context)

    // ===== 数据库 =====
    val database: AppDatabase = Room.databaseBuilder(
        context, AppDatabase::class.java, "adfeed.db"
    ).fallbackToDestructiveMigration().build()

    // ===== 数据源 =====
    private val localFileSource = LocalFileDataSource(context.assets)
    private val interactionDao = database.interactionDao()
    private val analyticsDao = database.trackEventDao()

    // ===== Repository =====
    val adRepository: AdRepository =
        AdRepositoryImpl(localFileSource, dispatcherProvider)
    val channelRepository: ChannelRepository =
        ChannelRepositoryImpl(localFileSource)
    val interactionRepository: InteractionRepository =
        InteractionRepositoryImpl(interactionDao, dispatcherProvider)
    val analyticsRepository: AnalyticsRepository =
        AnalyticsRepositoryImpl(analyticsDao, dispatcherProvider)

    // ===== AI 数据服务（可替换实现） =====
    // 开发期：LocalAIDataService 读 assets/ai_data.json
    // 完成期：RemoteAIDataService 调大模型 API + Room 缓存
    private val aiDataService: AIDataService = LocalAIDataService(context)

    // ===== AI 增强数据源（封装 AdRepository + AI 数据，直接产出 AIEnhancedAd） =====
    val aiAdRepository: AIAdRepository =
        AIAdRepositoryImpl(adRepository, aiDataService)

    // ===== 播放器池（全局单例） =====
    val playerPool: PlayerPool = PlayerPoolImpl(maxSize = 2)

    // ===== ViewModel 工厂（简化版，用函数代替 Factory 类） =====
    fun createFeedViewModel() = FeedViewModel(
        aiAdRepository, channelRepository, interactionRepository,
        analyticsRepository
    )
    fun createDetailViewModel(adId: String) = DetailViewModel(
        aiAdRepository, interactionRepository, analyticsRepository, adId
    )
    fun createStatsViewModel() = StatsViewModel(analyticsRepository)
    fun createSearchViewModel() = SearchViewModel(aiAdRepository)
}
```

### 2.6 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| Room 数据库首次创建失败 | `fallbackToDestructiveMigration()` + 重新创建 |
| AppContainer 重复创建 | Application 级别单例，`by lazy` 保证唯一 |
| 导航目标不存在 | Navigation Compile-safe + catch `IllegalArgumentException` |
| 全局未捕获异常 | `Thread.setDefaultUncaughtExceptionHandler` + Timber.e 记录 |

## 3. 模块2：core — 基础核心

### 3.1 模块职责

core 是项目的最底层模块，**不依赖任何其他模块**，提供：

- **本地文件读取工具**：从 `assets/` 读取 JSON 文件并解析
- **图片加载器配置**：Coil 全局配置（面向本地 `file:///android_asset/` 路径）
- **通用 UI 组件**：BaseFragment/BaseViewModel/BaseCardViewHolder、EmptyStateView、ErrorStateView
- **协程调度器抽象**：便于单元测试替换
- **日志**：Timber 封装
- **通用工具**：扩展函数、时间工具、Flow 防抖等

> **设计意图**：提供 "Android 开发的脚手架"，让其他模块不用重复写 Base 类。所有功能面向本地数据设计。

```kotlin
// ===== 基础 Fragment =====
abstract class BaseFragment<VB : ViewBinding, VM : BaseViewModel>(
    private val inflate: (LayoutInflater) -> VB
) : Fragment() {

    protected abstract val viewModel: VM
    protected lateinit var binding: VB

    // 模板方法：固定的生命周期流程
    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?, savedInstanceState: Bundle?
    ): View? {
        binding = inflate(layoutInflater)
        return binding.root
    }

    override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
        super.onViewCreated(view, savedInstanceState)
        initViews()       // 子类：初始化 View
        observeState()    // 子类：观察 ViewModel StateFlow
        initListeners()   // 子类（可选）：设置点击/滚动监听
    }

    abstract fun initViews()
    abstract fun observeState()
    open fun initListeners() {}
}

// ===== 基础 ViewModel =====
abstract class BaseViewModel : ViewModel() {

    // 统一的加载状态
    protected val _loadingState = MutableStateFlow<LoadingState>(LoadingState.Idle)
    val loadingState: StateFlow<LoadingState> = _loadingState.asStateFlow()

    // 统一的错误通道
    protected val _errorEvent = MutableSharedFlow<ErrorInfo>()
    val errorEvent: SharedFlow<ErrorInfo> = _errorEvent.asSharedFlow()

    protected fun handleError(throwable: Throwable, userMessage: String = "操作失败") {
        Timber.e(throwable, userMessage)
        _errorEvent.tryEmit(ErrorInfo(userMessage, throwable))
        _loadingState.value = LoadingState.Idle
    }
}

sealed class LoadingState {
    data object Idle : LoadingState()
    data object Loading : LoadingState()
    data object Refreshing : LoadingState()
    data class LoadingMore(val page: Int) : LoadingState()
}

data class ErrorInfo(
    val userMessage: String,
    val throwable: Throwable? = null
)

// ===== 基础 ViewHolder（带资源清理） =====
abstract class BaseCardViewHolder(
    private val binding: ViewBinding
) : RecyclerView.ViewHolder(binding.root) {
    abstract fun bind(item: AdItem, position: Int)
    open fun onRecycled() {}      // ViewHolder 回收时清理
    open fun onAttached() {}      // 进入可视区
    open fun onDetached() {}      // 离开可视区
}
```

### 3.5 核心流程

#### 本地文件读取流程

```
LocalFileReaderImpl.readAssetsJson<AdData>("ad_data.json")
  │
  ├─ 1. 打开 assets 文件
  │     val inputStream = assetManager.open("ad_data.json")
  │     val text = inputStream.bufferedReader().use { it.readText() }
  │     ├─ FileNotFoundException? → throw FileNotFoundError("ad_data.json")
  │     └─ 成功 → 继续
  │
  ├─ 2. Moshi 解析 JSON
  │     val adapter = moshi.adapter<AdData>(AdData::class.java)
  │     val result = adapter.fromJson(text)
  │     ├─ JsonDataException? → throw JsonParseError("ad_data.json", e)
  │     └─ 成功 → 返回 AdData 对象
  │
  └─ 3. 结果转换（DTO → Domain Model）
        AdMapper.toAdItems(result) → List<AdItem>
```

#### 图片加载流程（Coil 面向本地文件）

```
Coil ImageLoader 加载本地图片
  │
  │ 请求: imageView.load("file:///android_asset/images/ad_001.jpg")
  │       或 imageView.load(R.drawable.ad_001)  // res 内资源
  │       或 imageView.load(File("/data/data/.../cache/thumb_001.jpg"))
  │
  ├─ 1. 内存缓存命中 → 直接返回
  │
  ├─ 2. 磁盘缓存命中 → 解码返回
  │
  ├─ 3. 从 assets/res 读取 + 解码
  │     ├─ 加载中: Shimmer 动画占位
  │     ├─ 成功: 解码 → 写入磁盘缓存 + 内存缓存
  │     └─ 失败: 显示 errorDrawable（PlaceholderImage）
  │
  └─ 配置：内存 LRU ≤ 20% 可用内存，磁盘 ≤ 250MB, TTL 7天
```

### 3.6 异常处理

| 异常场景 | 处理策略 |
|---------|---------|
| assets 文件缺失 | `FileNotFoundError` → ViewModel 转为 UI 错误态 |
| JSON 格式错误 | `JsonParseError` → 日志记录 + 空数据降级 |
| 图片文件不存在 | Coil 自动展示 error 占位图 |
| 图片 OOM | Coil 自动降采样 + `isLowRamDevice` 检查缩小缓存 |
| Context 为 null | ViewModel 中避免直接引用 Context，通过 ResourceProvider 替代 |

## 4. 模块3：data — 数据层

### 4.1 模块职责

data 模块实现 Domain 层的 Repository 接口，**所有数据来源于本地文件（assets/*.json）和 Room 数据库**：

- 从 `assets/ad_data.json` 加载广告数据（含预生成的 AI 摘要/标签）
- 从 `assets/channels.json` 加载频道列表
- Room 数据库管理：互动状态（点赞/收藏）、AI 缓存、埋点数据
- 内存分页（广告 JSON 一次性加载到内存，按页返回）
- 跨页面状态同步（SharedFlow 广播互动事件）

> **设计意图**：数据层是核心功能与可选功能的基础。所有数据本地化使数据流清晰可追溯，便于验证。

### 4.6 核心流程

#### 数据加载流程（Cache-First）

```
AdRepositoryImpl.getFeed(channelId, page=1, pageSize=20)
  │
  ├─ 1. 检查内存缓存（首次加载为空）
  │     val cached = memoryCache[channelId]
  │     if (cached != null) → 直接从 cached 分页返回
  │
  ├─ 2. 首次加载：从 assets JSON 读取全部广告
  │     val allAds = localFileDataSource.loadAds()  // 一次性读取全部
  │     │
  │     ├─ 成功 → 按 channelId 过滤 + 按 createdAt 排序
  │     │         └─ 存入 memoryCache[channelId] = filteredAds
  │     │
  │     └─ 失败（文件缺失/JSON 解析错）
  │          └─ return Result.failure(AppError.FileNotFoundError)
  │
  ├─ 3. 分页返回
  │     val pageStart = (page - 1) * pageSize
  │     val pageItems = filteredAds.drop(pageStart).take(pageSize)
  │     val hasMore = pageStart + pageSize < filteredAds.size
  │     return Result.success(PageResponse(pageItems, page, pageSize, hasMore))
  │
  └─ 4. 加载更多（page > 1）
        → 直接从 memoryCache 分页返回（无额外 IO）
```

#### 互动状态同步流程

```
InteractionRepositoryImpl.toggleLike(adId)
  │
  ├─ 1. 从 Room 查询当前状态（若不存在则创建默认值）
  │     val current = interactionDao.getOrCreate(adId)
  │
  ├─ 2. 反转状态 + 更新计数
  │     val newState = current.copy(
  │       isLiked = !current.isLiked,
  │       likeCount = current.likeCount + if (!current.isLiked) 1 else -1
  │     )
  │
  ├─ 3. 持久化到 Room
  │     interactionDao.upsert(InteractionEntity.from(adId, newState))
  │
  ├─ 4. 通过 SharedFlow 广播（跨页面同步）
  │     _interactionEvents.emit(InteractionEvent(adId, LIKE, newState))
  │
  └─ 5. 返回新状态
```

| 异常场景 | 处理策略 |
|---------|---------|
| `ad_data.json` 缺失 | 构建时 `lint` 检查 + 运行时 `FileNotFoundError` → 展示错误态 |

## 7. 模块6：player — 播放器模块

### 7.1 模块职责

## 8. 模块7：ai — AI智能增强（可选模块）

### 8.1 模块职责

ai 模块是**可选模块**，为广告数据提供 AI 增强能力：

- 广告数据与 AI 数据**彻底分离**：`AdItem` 不含任何 AI 字段，AI 数据独立存储在 `assets/ai_data.json`

### 9.4 曝光追踪核心实现

```kotlin
class ExposureTracker(
    private val analyticsRepo: AnalyticsRepository,
    private val sessionManager: SessionManager
) {
    private val exposedInSession = mutableSetOf<String>()
    private val pendingExposures = mutableMapOf<String, Long>() // adId → 首次可见时间

    /** 由 FeedFragment 的滚动事件触发 */
    fun onItemVisible(adId: String, visibleRatio: Float, channelId: String) {
        if (adId in exposedInSession) return          // 已曝光
        if (visibleRatio < 0.5f) {
            pendingExposures.remove(adId)
            return
        }
        if (adId !in pendingExposures) {
            pendingExposures[adId] = SystemClock.elapsedRealtime()
        }
    }

    fun onItemInvisible(adId: String) {
        pendingExposures.remove(adId)
    }

    /** 每秒 tick（由 ViewModel 的定时协程驱动） */
    fun tick(channelId: String) {
        val now = SystemClock.elapsedRealtime()
        val confirmed = mutableListOf<String>()
        for ((adId, firstSeen) in pendingExposures) {
            if (now - firstSeen >= 1000L) confirmed.add(adId)
        }
        for (adId in confirmed) {
            pendingExposures.remove(adId)
            exposedInSession.add(adId)
            // 异步写入 Room（不阻塞 UI）
            analyticsRepo.track(TrackEvent(
                eventType = EventType.EXPOSURE,
                adId = adId,
                channelId = channelId
            ))
        }
    }
}
```

### 9.5 统计计算

```kotlin
class StatsCalculator(private val dao: TrackEventDao) {

    suspend fun computeGlobalStats(): GlobalStats {
        val allEvents = dao.getAll()
        val exposures = allEvents.count { it.type == EventType.EXPOSURE.name }
        val clicks    = allEvents.count { it.type == EventType.CLICK.name }
        val likes     = allEvents.count { it.type == EventType.LIKE.name }
        val favorites = allEvents.count { it.type == EventType.FAVORITE.name }
        val shares    = allEvents.count { it.type == EventType.SHARE.name }

        val ctr = if (exposures > 0) clicks.toFloat() / exposures else 0f

        val channelStats = allEvents
            .filter { it.channelId.isNotEmpty() }
            .groupBy { it.channelId }
            .mapValues { (_, events) ->
                val exp = events.count { it.type == EventType.EXPOSURE.name }
                val clk = events.count { it.type == EventType.CLICK.name }
                ChannelStats(it.key, exp, clk, if (exp > 0) clk.toFloat() / exp else 0f)
            }

        return GlobalStats(exposures, clicks, likes, favorites, shares, ctr, channelStats)
    }
}
```
